# AlphaEvolve: A coding agent for scientific and algorithmic discovery

> **FunSearch research line**: AlphaEvolve is the direct successor to [[romera-paredes-2023-funsearch]] (FunSearch, Nature 2023). Both use evolutionary LLM-guided program search with a formal evaluator. AlphaEvolve extends FunSearch to a broader range of problems (67+ mathematical and engineering challenges) and uses Gemini as the underlying LLM. Read alongside FunSearch for the full line of work.

## Abstract

"In this white paper, we present AlphaEvolve, an evolutionary coding agent that substantially enhances capabilities of state-of-the-art LLMs on highly challenging tasks such as tackling open scientific problems or optimizing critical pieces of computational infrastructure."

## TL;DR

AlphaEvolve extends FunSearch to 67+ mathematical and engineering problems using Gemini as the underlying LLM. Key mathematical result: discovers a procedure multiplying two 4×4 complex-valued matrices using 48 scalar multiplications — the first improvement over Strassen's algorithm for this setting after 56 years. Also optimizes Google data center scheduling, LLM training, and hardware accelerator circuits.

## Problem

FunSearch demonstrated evolutionary LLM search for mathematical discovery but was limited to one or two test problems (cap set, bin packing). Can the paradigm scale to a broad suite of mathematical and engineering optimization challenges, and can it discover genuinely new algorithms in multiple domains?

## Approach

Same core paradigm as [[romera-paredes-2023-funsearch]] — FunSearch extended:
- **LLM**: Gemini (stronger than FunSearch's original model).
- **Evaluator**: domain-specific formal evaluators for each of 67+ tasks.
- **Evolutionary loop**: propose code variants → evaluate → select best → use as context for next LLM call.
- **Broader scope**: spans pure mathematics, theoretical CS, and real-world engineering optimization.

Technical innovations over FunSearch (based on blog/white paper; full technical details may be in supplementary):
- Improved multi-prompt template management.
- Better handling of long-running evaluations.
- Gemini's code generation outperforms earlier LLMs on mutation quality.

## Key result

**Mathematics / algorithms:**
- **4×4 complex-valued matrix multiplication**: 48 scalar multiplications — **first improvement over Strassen after 56 years** for this problem variant. Note: this is distinct from AlphaTensor's [[fawzi-2022-alphatensor]] result (47 multiplications for 4×4 real-valued over Z₂) — different field and value type.
- **Kissing number problem**: improved the bound on the number of unit spheres that can touch a given sphere in specific dimensions.
- Mathematical analysis, combinatorics, geometry, number theory — improved or rediscovered best-known solutions across 67 problems.

**Engineering:**
- More efficient data center scheduling algorithm.
- Accelerated LLM training underpinning the system itself.
- Simplified hardware accelerator circuit design.

## Assumptions

- Gemini's code generation capability covers the program space relevant to the mathematical problems.
- The evaluators for each of 67 problems are sound (accept only correct solutions) and fast enough for the evolutionary loop.
- The evolutionary approach (without RL) is sufficient to find globally competitive solutions in many domains.

## Limitations / scope

- White paper format — full technical details (ablations, comparison methodology) may be in supplementary or future publications.
- Evolutionary search requires an efficient domain-specific evaluator — not all mathematical problems admit such evaluators.
- Engineering applications (data centers, LLM training) are more practically impactful but outside math-focused sub-area C scope.

## Replication evidence

Results and code at https://github.com/google-deepmind/alphaevolve_results. The 4×4 complex matrix multiplication result (48 multiplications) is mathematically verifiable independently.

## Why this paper matters

AlphaEvolve is the 2025 update demonstrating that the FunSearch paradigm scales:
1. **67 problems**: not one or two showcase cases but a diverse benchmark showing consistent performance.
2. **Genuine new algorithms**: the 4×4 complex matrix multiplication improvement is a new mathematical result (distinct from AlphaTensor's Z₂ result).
3. **Engineering + math together**: the same system discovers mathematical algorithms AND optimizes real infrastructure — demonstrating that the paradigm generalizes beyond pure math.

**Comparison with AlphaTensor ([[fawzi-2022-alphatensor]])**:
- AlphaTensor: RL + MCTS; specifically for tensor decomposition; foundational (2022).
- AlphaEvolve: evolutionary LLM search; broader domains; 2025 update.
- Both improve matrix multiplication algorithms but over different mathematical settings (real vs. complex, different fields).

**Cross-vault relevance**: AlphaEvolve's ability to discover new algorithms in combinatorics and mathematical analysis directly suggests applying the same paradigm to KB completion algorithm discovery for the Mixer. The evaluator for Mixer would be: "does this rule injection strategy reduce KB convergence time?" The evolutionary search would discover better injection heuristics than human-designed ones.
