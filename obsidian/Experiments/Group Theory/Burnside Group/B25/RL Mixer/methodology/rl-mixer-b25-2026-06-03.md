# Experiment — RL Mixer on B(2,5)

Training logs and evaluation output directories are not preserved. Dates of training: unknown.

## What this experiment is

Reinforcement learning applied to word reduction in B(2,5). Rather than applying KB rules in a fixed order (as in beam search), a PPO-trained agent learns a policy mapping word states to reduction strategy selections. The agent's action space encodes both *what strategy to use* and *where in the word to apply it*.

**Architecture:**
- **Rust core** (`src/`): reduction engine, Aho-Corasick pattern matching, beam search
  - `reduce/braid.rs` — braid shortening operations (CORRECT implementation — not the buggy version from b25_reduce_core)
  - `reduce/beam.rs` — split beam search (forward + inverse rules)
  - `actions.rs` — 80 macro actions (10 strategies × 8 position buckets)
- **Python bindings** (`python/rl_mixer/`): compiled .so via PyO3/maturin
- **RL training** (`rl/`): PPO implementation in pure Python using the Rust environment

**Action space**: 80 discrete actions = 10 reduction strategies × 8 position buckets (splits the word into 8 regions; each action applies a strategy focused on one region).

**Environment**: `rl_mixer.RlEnvironment('/tmp/all_rules_combined.live', 5)` — takes a rule bank file and a B(2,5) exponent (5). The environment wraps the Rust reduction engine.

## Hypothesis

A PPO-trained RL agent that selects among 80 macro reduction strategies can outperform the fixed-order beam search on B(2,5) word reduction (shorter reduced word length for the same wall-clock budget, or same length in less time).

## Problem set

- Group: B(2,5), free Burnside group on 2 generators of exponent 5
- Target words: B(2,5) commutator words, specifically:
  - `comm_12_7` — commutator [c12, c7] (confirmed from checkpoint name `subword_comm12_7.pt`)
  - `comm_16_2` — commutator [c16, c2] (confirmed from `subword_comm16_2.pt`)
  - RPO-rule-scored target (from `subword_rpo.pt`)
- Input rule bank: `/tmp/all_rules_combined.live` (large combined B(2,5) rule bank; not on disk — external runtime dependency)

- RL word reduction validity: each reduction step applies a valid B(2,5) rule, so any reduced word is a valid B(2,5) identity consequence. The RL policy selects *which* valid rules to apply, not whether they are valid.
